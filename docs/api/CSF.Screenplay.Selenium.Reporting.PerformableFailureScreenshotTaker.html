Remarks

A screenshot is only taken if the Exception is not an instance of PerformableException and if the Actor has the ability BrowseTheWeb. If the exception is a performable exception then this indicates that an 'inner' performable is the true source of the exception and that this failure simply represents an exception bubbling upward through the performable stack. Additionally, if the actor cannot browse the web, then they of course cannot possibly take a screenshot.

This handler internally uses the task TakeAndSaveScreenshot and configures it to fail silently if taking a screenshot is not supported by the current browser. Thus, this is not certain to always result in a screenshot. When a screenshot is taken in this way, it is saved alongside the report as an asset file.
